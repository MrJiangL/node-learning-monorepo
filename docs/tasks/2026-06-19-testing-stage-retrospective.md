# Task: 测试强化阶段复盘

## 背景

你刚刚连续练了三层测试：

```text
Service 单元测试
Repository 测试
API 集成测试
```

这张任务不写业务代码，目标是把这三层测试在你脑子里放到正确的位置。

测试不是为了“写得越多越好”，而是为了知道：

```text
这段代码最怕坏在哪里？
我应该在哪一层用最低成本把它保护住？
```

---

## 这张任务只练什么

只写复盘文档：

```text
docs/reviews/testing-stage-retrospective.md
```

暂时不新增接口、不改 Prisma、不补复杂测试。

---

## 学习目标

完成后你应该能说清楚：

```text
1. Service 测试主要保护什么
2. Repository 测试主要保护什么
3. API 集成测试主要保护什么
4. 什么测试是必须写的，什么测试可以先不写
5. 你现在最卡的是测试设计、mock、数据准备，还是断言
```

---

## 任务 1：创建复盘文档

创建：

```text
docs/reviews/testing-stage-retrospective.md
```

写下面几个小标题：

```md
# 测试强化阶段复盘

## 1. Service 测试在测什么

## 2. Repository 测试在测什么

## 3. API 集成测试在测什么

## 4. 什么情况下测试是必须写的

## 5. 什么情况下可以先不写测试

## 6. 我现在写测试最卡的地方

## 7. 下一阶段我想继续学什么
```

每节 2-5 句即可，不需要写得很正式。

---

## 任务 2：用自己的话回答“测试是不是必须写”

你可以按这个角度写：

```text
不是每一行代码都必须写测试。
但是关键业务规则、权限边界、数据写入、错误处理、复杂查询，最好有测试保护。
```

你也可以写你自己的真实感受，比如：

```text
我现在能看懂测试，但还不太会自己设计测试场景。
```

这个阶段允许这样写，因为我们要先把“测试判断力”练出来。

---

## 任务 3：选择下一阶段方向

在文档最后选一个方向：

```text
A. 继续后端工程化：配置、日志、监控、部署准备
B. 深入数据库：索引、查询优化、事务、并发
C. 深入测试：mock、factory、integration、coverage
D. 前后端联调体验：把 Activity Log 接入 Vue 页面
```

我建议你选：

```text
B. 深入数据库
```

原因：

```text
你现在已经有 API、鉴权、缓存、后台任务、Activity Log。
下一步如果补数据库索引、查询优化和事务边界，会更像真实后端开发。
```
