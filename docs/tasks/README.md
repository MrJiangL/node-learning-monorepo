# 学习任务协作方式

这个目录放每日/每阶段的练习任务。我们的节奏固定成下面这样。

## 开发节奏

1. 我先写一份任务 Markdown。
2. 你按任务自己实现代码。
3. 你完成后告诉我“完成了”。
4. 我负责跑测试、类型检查、格式检查和构建。
5. 如果有问题，我指出是哪一层没串起来，并解释原因。
6. 如果功能通过，我再帮你补充学习型注释，让代码更适合复习。
7. 我最后给你下一张任务卡。

## 你每次实现时先做什么

每张任务卡都会先让你写测试，或者我已经提前帮你写好失败测试。你实现前先跑一次测试，确认它确实失败。

常用命令：

```bash
npm run test
npm run typecheck
npm run format
npm run format:check
```

## 任务卡约定

任务卡会包含：

- 目标：这次练什么能力。
- 涉及文件：你需要打开哪些文件。
- RED：先写或运行失败测试。
- GREEN：你要实现的代码方向。
- 注释版代码提示：每段关键代码都会有学习型注释。
- 自查清单：你完成后自己先检查一遍。

任务卡里的代码片段是学习提示，不是让你无脑复制。你可以照着敲，但要边敲边理解每一层为什么存在。
