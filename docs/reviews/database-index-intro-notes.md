# 数据库索引入门笔记

## 1. 索引是干嘛的

索引用来帮助数据库更快定位数据，避免每次查询都扫描整张表。

可以粗略理解成书的目录：没有目录时只能一页页找，有目录时可以先定位到大概范围。

## 2. 单字段索引和复合索引的区别

单字段索引只针对一个字段，比如 `@@index([action])`。

复合索引把多个字段按顺序放在一个索引里，比如 `@@index([userId, projectSnapshotId, createdAt])`。

复合索引不是简单地“多个单字段索引加起来”，它更适合服务多个条件一起出现的查询。

## 3. Activity Log 为什么适合补 userId + projectSnapshotId + createdAt 索引

Activity Log 列表查询的核心条件是：

- `userId`：保证当前用户只能查自己的日志
- `projectSnapshotId`：定位某个 Project 的历史日志
- `createdAt`：支持按时间倒序列表和时间范围过滤

这三个字段会在同一个查询里一起出现，所以适合补一个复合索引。

这样数据库可以先按用户缩小范围，再按 Project 快照缩小范围，最后按时间处理列表和过滤。

## 4. 为什么索引不是越多越好

索引可以提升某些查询速度，但不是越多越好。

因为新增、更新、删除数据时，数据库不只要改表数据，还要维护相关索引，所以写入会更慢。

索引还会占用额外存储空间；如果索引太多，也会增加后续维护和判断成本。

## 5. 我现在还不懂的地方

我现在还不太懂怎么判断一个旧索引是不是已经没用了。

比如有了 `userId + projectSnapshotId + createdAt` 之后，原来的 `userId + createdAt`、`projectSnapshotId + createdAt` 是否还要保留，需要继续通过真实查询和索引规则判断。
